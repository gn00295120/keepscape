# Judging evidence map

This file keeps implementation priorities tied to the four equally weighted Build Week criteria.

| Criterion | Claim | Evidence judges can inspect |
| --- | --- | --- |
| Technological implementation | GPT-5.6 emits a strict source-linked blueprint containing no geometry; the host compiles a safe diorama; Codex returns a typed interaction plus `{preset, orderedPhotoSourceIds}`; the host applies canonical slots and validates allowlists, references, and schema. | `/api/blueprint`, `/api/build`, `createPhotoDiorama`, exact photo-set and hotspot checks, post-build schema test, actual trace and receipt. |
| Design | Keepscape is a complete archive-to-walkable-space journey, not a prompt box or isolated demo. | Source desk, uncertainty review, build transition, multi-photo diorama, Evidence Lens, second mechanic, responsive and keyboard paths. |
| Potential impact | Families can preserve the context behind ordinary photos without surrendering truth to generative fill-in. | Original-voice timecodes, cited photo views, manually calibrated fixture regions, text-only notes that never masquerade as audio, typed human decisions, generated-language review, no-cloning policy. |
| Quality of idea | Generated space and provenance coexist inside the emotional experience. | Evidence Lens preserves the selected cited source view and, when reviewed metadata exists, outlines its region; Lantern and Repair share one contract but produce different play. |

## Demo proof order

1. Show the scattered source problem.
2. Show GPT-5.6 linking claims to evidence and exposing uncertainty.
3. Show the human resolving uncertainty and approving the displayed claims and generated story copy.
4. Show Codex returning a typed interaction and bounded photo-order plan, then the host's structural checks and receipt.
5. Walk into the first exhibit, turn on Evidence Lens, and open a source card at the emotional peak.
6. Play enough of the second mechanic to prove the first was not a template.
7. Close on a real product promise, not a feature list.

The public judge path is a verified deterministic replay with its fallback trace shown in-product. It proves the
runtime and validation journey without implying that GPT-5.6 or Codex ran live during that browser session.
