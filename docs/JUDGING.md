# Judging evidence map

This file keeps implementation priorities tied to the four equally weighted Build Week criteria.

| Criterion | Claim | Evidence judges can inspect |
| --- | --- | --- |
| Technological implementation | GPT-5.6 converts multimodal evidence into a typed, source-linked blueprint; Codex creates a bespoke typed interaction that the host verifies. | `/api/blueprint`, `/api/build`, schema reference checks, actual run trace, build receipt, unit tests. |
| Design | Keepscape is a complete archive-to-exhibit journey, not a prompt box or isolated demo. | Source desk, uncertainty review, build transition, two playable mechanics, source drawer, responsive and keyboard paths. |
| Potential impact | Families can preserve the context behind ordinary photos without surrendering truth to generative fill-in. | Original-voice timecodes, photo-region provenance, human confirmation, no-cloning policy, offline deterministic exhibits. |
| Quality of idea | Generated interaction and provenance coexist inside the emotional experience. | Lantern collection and repair-sequence samples share one contract but produce materially different play; interpretation is visibly separated from fact. |

## Demo proof order

1. Show the scattered source problem.
2. Show GPT-5.6 linking claims to evidence and exposing uncertainty.
3. Show the human confirmation boundary.
4. Show Codex building, testing, and returning a receipt.
5. Play the first exhibit and open a source card at the emotional peak.
6. Play enough of the second mechanic to prove the first was not a template.
7. Close on a real product promise, not a feature list.
